Authentication Endpoints

POST /api/auth/register/

Register a new user account. Returns user details and JWT tokens.

{
  "username": "aisha",
  "email": "aisha@example.com",
  "password": "password123",
  "password_confirm": "password123",
  "first_name": "Aisha",
  "last_name": "Omar"
}
POST /api/auth/token/

Login user and receive JWT access and refresh tokens.

{
  "username": "aisha",
  "password": "password123"
}
POST /api/auth/token/refresh/

Refresh an expired access token using a valid refresh token.

{
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
GET /api/auth/profile/

Retrieve the authenticated user's profile information. Requires authentication.

Activities Endpoints

GET /api/activities/

List all activities for the authenticated user. Requires authentication.

POST /api/activities/

Create a new activity. Requires authentication.

{
  "type": "running",
  "duration": 30,
  "calories": 250,
  "distance": 5.2,
  "date": "2025-08-29",
  "notes": "Evening run around the park"
}
GET /api/activities/<id>/

Retrieve a specific activity by ID. Requires authentication and ownership.

PUT /api/activities/<id>/

Update a specific activity. Requires authentication and ownership.

DELETE /api/activities/<id>/

Delete a specific activity. Requires authentication and ownership.

Testing Instructions

1. Start the development server:

python manage.py runserver 0.0.0.0:8080

2. Register a new user:

curl -X POST http://127.0.0.1:8080/api/auth/register/ \
-H "Content-Type: application/json" \
-d '{
  "username": "testuser",
  "email": "test@example.com",
  "password": "password123",
  "password_confirm": "password123",
  "first_name": "Test",
  "last_name": "User"
}'

3. Login to get JWT tokens:

curl -X POST http://127.0.0.1:8080/api/auth/token/ \
-H "Content-Type: application/json" \
-d '{
  "username": "testuser",
  "password": "password123"
}'

4. Use the access token for authenticated requests:

curl -X GET http://127.0.0.1:8080/api/activities/ \
-H "Authorization: Bearer YOUR_ACCESS_TOKEN_HERE"
